# ADR 0005: ClickHouse для аналитического хранилища

## Статус
Принято.

## Решение
Выбран ClickHouse 24.3 как колоночная СУБД для real-time аналитики.

## Обоснование
- Kafka Engine для прямого чтения из топиков
- Materialized Views для агрегации на лету
- Высокая производительность на аналитических запросах
- Нативная интеграция с Grafana

## Альтернативы
- Druid: сложнее в настройке
- TimescaleDB: хуже для аналитики
