# ADR 0004: ksqlDB вместо Apache Flink

## Статус
Принято.

## Решение
Выбран ksqlDB 0.29 для потоковой агрегации.

## Обоснование
- SQL-интерфейс — минимальный порог входа
- Нативная интеграция с Kafka
- Достаточно для оконных агрегаций курсового проекта
- Меньше boilerplate чем PyFlink

## Альтернативы
- Flink: мощнее, но требует Java/Scala или PyFlink с большим overhead
