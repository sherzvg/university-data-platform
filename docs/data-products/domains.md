# Домены данных платформы

| Домен | Назначение | Источники | Потребители |
|---|---|---|---|
| `academic_performance` | Оценки, посещаемость, контрольные | LMS API (Moodle-симулятор), CSV из деканата | ML-модель прогноза отчисления, дашборд успеваемости |
| `campus_infrastructure` | Загруженность аудиторий, RFID-проходы | Kafka-поток событий со СКУД, расписание (CSV) | Real-time дашборд, отчёт по утилизации |
| `student_engagement` | Заходы в LMS, сдача заданий, форумы | События Kafka из LMS, выгрузки активности | Скоринг вовлечённости |

## Принципы Data Mesh
- Каждый домен — отдельный Data Product со своим владельцем.
- Контракты схем версионируются (JSON-Schema в `contracts/`).
- SLA фиксируются в `docs/data-products/<domain>.md`.
- Изменения схемы — через PR + ADR.
